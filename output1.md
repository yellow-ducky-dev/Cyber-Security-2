Executive Summary</strong></h1><p>This report documents a three-week cybersecurity internship task involving the security assessment and hardening of a Node.js web application. The original application (vuln-nodejs-app) contained multiple critical vulnerabilities. This report covers the identification of those vulnerabilities (Week 1), the implementation of security fixes (Week 2), and advanced security testing with logging and final documentation (Week 3).</p><h1><strong>2. Week 1: Security Assessment</strong></h1><h2><strong>2.1 Application Overview</strong></h2><p>Application: payatu/vuln-nodejs-app (GitHub) — a purposely vulnerable Node.js/Express web application.</p><p>Tech stack: Node.js, Express, EJS templating, Sequelize ORM, MySQL, MongoDB, JWT.</p><p>Pages assessed: Login, Register, Profile, XSS Lab, Ping Lab, File Upload.</p><h2><strong>2.2 Vulnerability Assessment Methodology</strong></h2><ul><li>OWASP ZAP: Automated scan for common vulnerabilities (XSS, injection, misconfigured headers).</li><li>Browser Developer Tools: Manual XSS testing using &lt;script&gt;alert('XSS');&lt;/script&gt; in text fields.</li><li>SQL Injection: Tested login with: admin' OR '1'='1 in username/password fields.</li><li>Manual Code Review: Inspected controllers, routes, and models for logic flaws.</li></ul><h2><strong>2.3 Vulnerabilities Found</strong></h2><table><tr><td><p><strong>Vulnerability</strong></p></td><td><p><strong>Severity</strong></p></td><td><p><strong>Location</strong></p></td><td><p><strong>Description</strong></p></td></tr><tr><td><p>MD5 Password Hashing</p></td><td><p>CRITICAL</p></td><td><p>authController.js</p></td><td><p>Passwords stored as MD5 — easily crackable with rainbow tables.</p></td></tr><tr><td><p>Reflected XSS</p></td><td><p>HIGH</p></td><td><p>xss_lab route</p></td><td><p>URL query params rendered directly in EJS without sanitization.</p></td></tr><tr><td><p>Command Injection</p></td><td><p>HIGH</p></td><td><p>ping_get controller</p></td><td><p>User input passed directly to exec() shell command.</p></td></tr><tr><td><p>JWT No Expiry</p></td><td><p>MEDIUM</p></td><td><p>generateAccessToken()</p></td><td><p>Tokens issued with no expiration — stolen tokens valid forever.</p></td></tr><tr><td><p>No Rate Limiting</p></td><td><p>MEDIUM</p></td><td><p>All auth routes</p></td><td><p>Login endpoint allows unlimited brute-force attempts.</p></td></tr><tr><td><p>Missing Security Headers</p></td><td><p>MEDIUM</p></td><td><p>server.js</p></td><td><p>No Helmet.js — missing CSP, X-Frame-Options, HSTS headers.</p></td></tr><tr><td><p>Verbose Error Messages</p></td><td><p>LOW</p></td><td><p>Login route</p></td><td><p>Error reveals whether username or password is incorrect.</p></td></tr><tr><td><p>No Request Logging</p></td><td><p>LOW</p></td><td><p>server.js</p></td><td><p>No audit trail for security events or failed login attempts.</p></td></tr></table><h1><strong>3. Week 2: Security Hardening</strong></h1><h2><strong>3.1 Password Hashing — bcrypt</strong></h2><p>Problem: Passwords were hashed with MD5 — a fast, collision-prone algorithm not suitable for passwords.</p><p>Fix: Replaced MD5 with bcrypt (cost factor = 10), which is slow by design and includes a salt automatically.</p><p>Library: npm install bcrypt</p><h2><strong>3.2 Input Sanitization &amp; Validation</strong></h2><p>Problem: User inputs (username, email) were not validated or sanitized before use, enabling XSS and injection.</p><p>Fix: Used the validator library to validate email format, enforce alphanumeric usernames (3–30 chars), and escape HTML characters with validator.escape().</p><p>Library: npm install validator</p><h2><strong>3.3 JWT Authentication with Expiry</strong></h2><p>Problem: JWT tokens were issued without an expiration time, meaning stolen tokens were valid indefinitely.</p><p>Fix: Added { expiresIn: '1h' } to jwt.sign() and implemented a Bearer token middleware for protected routes.</p><p>Library: npm install jsonwebtoken</p><h2><strong>3.4 HTTP Security Headers — Helmet.js</strong></h2><p>Problem: The application lacked HTTP security headers, making it vulnerable to clickjacking, MIME sniffing, and XSS via injected scripts.</p><p>Fix: Applied helmet() middleware and configured a strict Content Security Policy (CSP) allowing only same-origin scripts.</p><p>Library: npm install helmet</p><h2><strong>3.5 Rate Limiting</strong></h2><p>Problem: No rate limiting on auth endpoints allowed unlimited brute-force login attempts.</p><p>Fix: Applied express-rate-limit with a window of 15 minutes / 10 attempts on all /api/auth/* routes. Violations are logged.</p><p>Library: npm install express-rate-limit</p><h1><strong>4. Week 3: Advanced Testing &amp; Logging</strong></h1><h2><strong>4.1 Penetration Testing Simulation</strong></h2><ul><li>XSS Test: Entered &lt;script&gt;alert('XSS');&lt;/script&gt; in all input fields. Result: BLOCKED — validator.escape() encoded the input as &amp;lt;script&amp;gt;...</li><li>SQL Injection: Tested admin' OR '1'='1 in username field. Result: BLOCKED — parameterized queries via Sequelize prevent injection.</li><li>Brute Force: Sent 11 rapid login requests. Result: BLOCKED — rate limiter returned HTTP 429 on 11th attempt.</li><li>Expired Token: Waited for token expiry and accessed /profile. Result: BLOCKED — middleware returned HTTP 403.</li></ul><h2><strong>4.2 Security Logging — Winston</strong></h2><p>Implemented Winston logger with two transports: Console (development) and File (logs/security.log).</p><p>All HTTP requests, failed authentications, rate limit violations, and unhandled errors are logged with timestamps.</p><p>Library: npm install winston</p><h2><strong>4.3 Security Best Practices Checklist</strong></h2><table><tr><td><p><strong>Best Practice</strong></p></td><td><p><strong>Status</strong></p></td><td><p><strong>Tool Used</strong></p></td></tr><tr><td><p>Validate and sanitize all user inputs</p></td><td><p>DONE</p></td><td><p>validator</p></td></tr><tr><td><p>Hash and salt passwords</p></td><td><p>DONE</p></td><td><p>bcrypt</p></td></tr><tr><td><p>Use token-based authentication with expiry</p></td><td><p>DONE</p></td><td><p>jsonwebtoken</p></td></tr><tr><td><p>Set secure HTTP headers</p></td><td><p>DONE</p></td><td><p>helmet.js</p></td></tr><tr><td><p>Rate limit authentication endpoints</p></td><td><p>DONE</p></td><td><p>express-rate-limit</p></td></tr><tr><td><p>Log security events and errors</p></td><td><p>DONE</p></td><td><p>winston</p></td></tr><tr><td><p>Use HTTPS for data transmission</p></td><td><p>RECOMMENDED</p></td><td><p>TLS cert + NGINX</p></td></tr><tr><td><p>Store secrets in environment variables</p></td><td><p>DONE</p></td><td><p>.env / process.env</p></td></tr><tr><td><p>Use generic error messages</p></td><td><p>DONE</p></td><td><p>authController.js</p></td></tr><tr><td><p>Limit request body size</p></td><td><p>DONE</p></td><td><p>10kb limit in express</p></td></tr></table><h1><strong>5.
